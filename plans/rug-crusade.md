# The rug-crusade — cold-start spec (2026-07-20)

The free, useful, viral anti-rug tool that gathers the aggrieved RH-Chain tribe (the
band). It's the top of the funnel that *starts* the whole flywheel: **movement → band
→ community-owned launchpad → self-funding growth (Bounties in veJOHN).**

## The hook is utility + mission — NOT points
Points appetite is weak for a small project on an uncertain chain (fatigue; farmers
chase big credible airdrops, not uncertain points). So **do not run a points
program** — it attracts mercenaries and signals desperation. The rug-crusade pulls on:
1. **Utility** — "don't get rugged." People use it because it *saves them money.*
2. **Mission** — the band vs the ruggers. Belonging + shared enemy + shared grievance
   (63% of RH traders are underwater; everyone's been scammed). Robin Hood's exact
   people, currently unorganized.
3. **Virality** — rug drama spreads itself. Warnings + exposés get shared hard.

**Reward the early band RETROACTIVELY at TGE** — a surprise "you were here early" veJOHN
airdrop, weighted by genuine contribution (rugs reported, alerts spread, members
recruited), tracked quietly. Retro beats announced-points: it rewards believers not
farmers, and the surprise builds goodwill. You can build the whole band on mission +
utility *before* any token exists.

## The product (v1)
1. **Rug/scam scanner** — paste a token → an "am I safe?" verdict: honeypot (sell
   simulation), concentrated bid-wall (the based-hood exploit class), serial-rugger
   deployer, LP-lock/burn status, mint authority, holder concentration. Off the
   indexer + on-chain reads + heuristics.
2. **Real-time rug alerts** — auto-detect + push *"🚨 $X just rugged / is a honeypot —
   deployer flagged"* to a **Telegram bot** (in the trenches groups) + a **Twitter
   account** (auto-post). The alerts *are* the content and the distribution.
3. **The wall of shame** — public running leaderboard of serial ruggers + scam
   deployers (wallet clusters). Shareable, recurring, names the enemy.
4. **Deployer reputation** — every token page shows its deployer's history (prior
   rugs?). This is the data that later feeds the community-governed launchpad's
   blacklist — the recruiting tool *is* the input to the product.

## Delivery — go where they already are
- **TG bot** added to the big RH-Chain trading groups (admins *want* a rug-checker for
  their members) — zero-cost distribution parasitic on communities that already exist.
- **Twitter** auto-posting the alerts + wall-of-shame updates — the founder voice,
  automated, so it doesn't depend on you being a full-time influencer.
- A **section of littlejo.hn** (the web app) as the home base.

## The flywheel connection
rug-crusade (utility+mission → the band) → the band **governs the launchpad**
(blacklist ruggers, vet launches — the novel wedge) → the owned launchpad monetizes
(the take) → **Bounties reward the band in veJOHN** (`bounties-engine.md`) → growth
compounds. The rug-crusade is the free, cheap, on-brand thing that spins it up.

## Reuses (already built)
Indexer (rug-detection data) · the security chops (from auditing our own contracts +
dissecting based-hood's drain) · app + edge infra · firehose (real-time alerts). New
to build: the heuristics engine (honeypot sim, bid-wall detection, serial-rugger
clustering, LP-lock check), the TG bot + Twitter auto-post pipeline, the wall-of-shame
+ deployer-reputation UI.

## Honest risks
1. **False-positive rug-calls = credibility + legal risk.** Lead with *data/receipts*,
   flag *"risk"* not *"fraud."* Be the careful sheriff, not the mob. This is the #1
   thing that can kill the brand.
2. **You still have to seed it + be present** in the groups early — the data carries
   most of it, but not all. It's cheap, not zero-effort.
3. **False-negatives too** — sophisticated rugs slip past heuristics. Don't promise
   "safe"; promise "we flag the obvious traps." Over-promising safety is a trust bomb.
4. **Still bets on RH Chain** surviving the September cliff. If the chain dies, the
   tribe scatters.
5. **Retro-reward, not a promise** — never *promise* an airdrop (securities + it
   re-mercenary-izes it); do it as a genuine surprise so it stays a movement, not a farm.
